# ADR 0038 — Expression contract for AI-authored code

- **Status:** Accepted
- **Date:** 2026-05-02
- **Deciders:** S-0026 (Phase 2 → Phase 3 bridge — code-discipline contract authoring before substantive build phases author Python at scale)

## Context

The project carries two expression contracts. [ADR 0027](../../product/adr/0027-rendering-policy-prompt-layer-contract.md) governs learner-facing prose through a forbidden-token enumeration with positive-whitelist surviving tokens; its load-bearing surface is the prohibition list, and tightening (adding a forbidden token) is cheap while loosening is expensive. [ADR 0036](0036-expression-contract-for-inward-documents.md) governs inward-facing project documentation through a positive voice characterization audited by the present-state worked example in [`../operations/document-voice.md`](../operations/document-voice.md); its load-bearing surface is the positive description, and refinement is cheap while posture change is expensive. Each contract pairs ADR (the citable contract) with operational document (the working surface); each is calibrated to what its surface protects.

A third expressive surface — code authored by the AI for the build apparatus — has had no comparable contract. The current code surface is small ([`../tools/validate.py`](../tools/validate.py) at 457 lines plus shell scripts under [`../tools/`](../tools/) and [`../tools/hooks/`](../tools/hooks/)), but [Phase 4 (graph construction)](../../build_plan/P_2_graph_validation.md) per [ADR 0016](0016-graph-construction-needs-live-validation.md) is the inflection point at which Python authoring volume grows — the live graph-audit utility flesh-out is substantial, and every downstream phase (Phase 5 seed-graph build, Phase 6 self-correction, Phase 7 teaching layer, Phase 8 evaluation) authors Python or TypeScript against the schema.

The audit channel a code contract must compensate for differs structurally from the prose channel. Inward-facing prose is read by the user during authoring and shutdown; the human eye catches voice drift directly because the artifact's value *is* the prose. Code's value is the behavior under all inputs, not the lines as written; reading line-by-line does not reveal compound drift — the failure mode where a small wrong premise about an invariant, a return shape, a precondition is built upon consistently across many lines, producing internally coherent code that passes tests authored against the same wrong premise. The user cannot reliably audit code at scale, and the prose-style discipline that works for ADR 0036 does not transfer.

The three-layer structure this ADR commits to compensates for the weaker audit channel by mechanizing what mechanization can catch (low-level type, format, and test drift) and by forcing the auditable layer up into prose (the per-artifact contract block) and into a fresh-eyes review pass (the cold-context review at shutdown). Tests are necessary but insufficient for the compound-drift mode; tests authored under the same wrong premise pass. Mechanical gates are necessary but insufficient; type-check and lint cleanliness are silent on whether the code solves the right problem. The contract block in prose and the cold-context review at shutdown are what targets premise drift specifically.

## Decision

The project authors a third expression contract — distinct from [ADR 0027](../../product/adr/0027-rendering-policy-prompt-layer-contract.md)'s rendering policy and [ADR 0036](0036-expression-contract-for-inward-documents.md)'s inward-document contract — governing AI-authored code under [`../`](../) (the engine subtree). The operational surface lives at [`../operations/code-discipline.md`](../operations/code-discipline.md). This ADR is the citable contract.

The contract's posture: code under engine/ is authored under three discipline layers that compose. Each layer targets a different failure mode; together they compensate for the weak human-audit channel.

**Layer 1 — Contract-first prose.** Before non-trivial code is authored, the AI writes a contract block that names purpose, inputs, outputs, invariants the code maintains, what the code does NOT do, and edge cases. The form depends on scope: function and class scope use a docstring contract in the file's existing docstring style; module-level decisions (data structures, public APIs, integration boundaries) and cross-cutting design choices (file layout, dependency choices) use an ADR. The user audits this layer as prose; code is judged against its contract, not against the AI's intentions.

**Layer 2 — Mechanical gates.** The pre-commit hook gates any modified Python file under engine/ on lint cleanliness, format cleanliness, strict type-check cleanliness, and test execution. The specific tools — `ruff` for lint and format, `mypy --strict` for types, `pytest` for tests — are enumerated in [`../operations/code-discipline.md`](../operations/code-discipline.md) so that adding a tool to the stack (a security scanner, a complexity check) is a refinement under this contract rather than a posture change. The gates do not validate the meaning of the code; they make low-level drift visible cheaply.

**Layer 3 — Cold-context review pass.** At session shutdown, if the session modified tracked code under engine/, the shutdown sequence launches a sub-agent review pass with no session context. The agent reads the diff and the relevant contract blocks and reports whether the code matches its contract. Findings are recorded in [`../session/current.json`](../session/current.json)'s `outcome_summary` before archive. Fresh eyes that do not share the authoring AI's premises are the analog to the human spot-check that [ADR 0036](0036-expression-contract-for-inward-documents.md) relies on — the closest available channel for catching premise drift.

The contract's scope governs all Python code authored under [`../`](../). Shell scripts under [`../tools/`](../tools/) and [`../tools/hooks/`](../tools/hooks/) are exempt — they are short wrappers whose risk surface is bounded by their existing line count, and the gate stack does not yet include shell linting (refining the contract to add a shell-lint gate is a future ENGINE_LOG-tracked refinement). Generated artifacts ([`../tools/validate-history.jsonl`](../tools/validate-history.jsonl) and the like) are exempt by purpose. Product code — under [`../../product/`](../../product/) when it lands, likely Deno/TypeScript Supabase edge functions per [ADR 0029](../../product/adr/0029-personal-financial-cost-ceiling.md) and the runtime architecture in [`../../product/docs/architecture.md`](../../product/docs/architecture.md) — is governed by the same three-layer posture; the language-specific gate stack adapts at the session that lands the first product code (refinement, not posture change).

The contract's amendment discipline matches [ADR 0036](0036-expression-contract-for-inward-documents.md)'s structural pattern — refinement is cheap because it preserves the discipline; posture change is expensive because it is what the discipline binds. Refinements are ENGINE_LOG-tracked: sharpening contract-block conventions in the operational doc, adding a tool to the gate stack, refining the cold-review prompt template, expanding the per-language gate appendix when product code lands. Substantive changes to the posture itself require a superseding ADR: removing or restructuring any of the three layers, dropping the type-check gate as a category, abandoning the cold-review pass as a shutdown step. The asymmetry holds because the load-bearing surface is the three-layer commitment; tools and conventions sit beneath it.

## Consequences

- **The operational document is the working surface; this ADR is the citable contract.** [`../operations/code-discipline.md`](../operations/code-discipline.md) carries the contract-block authoring guidance, the worked examples of good and bad contract blocks, the gate-stack invocation commands, and the cold-review prompt template. Refinements land there under ENGINE_LOG tracking; posture changes require superseding this ADR. The same separation [ADR 0036](0036-expression-contract-for-inward-documents.md) maintains between itself and [`../operations/document-voice.md`](../operations/document-voice.md), and that [ADR 0027](../../product/adr/0027-rendering-policy-prompt-layer-contract.md) maintains between itself and [`../../product/AGENT_INSTRUCTIONS.md`](../../product/AGENT_INSTRUCTIONS.md).

- **Validation extension is core to this contract.** [ADR 0036](0036-expression-contract-for-inward-documents.md) names "validation extension is out of scope" — mechanical detection of voice violations against a positive characterization is unreliable, and authoring sessions are the appropriate enforcement surface for that contract. This contract inverts the relationship: mechanical gates are Layer 2 of the discipline, not an out-of-scope concern. The code surface admits mechanical detection of a category of drift (type, format, test) that prose does not; relying on authoring-session enforcement alone — the posture that works for prose — leaves the user's weakest audit channel uncovered.

- **`validate.py` is foundation; this session retrofits it under the contract.** [`../tools/validate.py`](../tools/validate.py) (457 lines, no tests at the time of this ADR's authoring) is what enforces the new gates on every commit. Building gate machinery on top of an unverified validator would reproduce compound drift at the infrastructure layer. The S-0026 session that authors this ADR also authors per-function contract blocks for `validate.py`, writes the pytest suite covering hard-fail and soft-warn paths, and brings the file to `mypy --strict` and `ruff` cleanliness — so the validator binds to its own gates from the first commit of this contract. No grandfathering; the contract applies uniformly from acceptance.

- **The cold-review pass adds a shutdown-sequence step.** [`../operations/session-shutdown-sequence.md`](../operations/session-shutdown-sequence.md) light-revises in this commit to insert the cold-review pass between spot-check (existing step 2) and STATE.md update (existing step 3) — sessions that modified tracked Python code under engine/ launch a sub-agent review against the modified files' contract blocks and record findings in `outcome_summary`. Sessions that authored prose only continue to skip the step.

- **The pre-commit hook extends to invoke the gate stack.** [`../tools/hooks/pre-commit`](../tools/hooks/pre-commit) gains a Python-files branch that runs `validate.py`'s `validate_code_gates()` when the staged diff includes any `engine/**/*.py` file. The gate stack runs only in build mode — exploration mode forbids Python edits anyway per [`../operations/session-build-lifecycle.md`](../operations/session-build-lifecycle.md).

- **Tooling enters the project.** A `requirements.txt` (or equivalent dependency manifest, location settled by the implementing session) declares `ruff`, `mypy`, `pytest` as engine-tool dependencies. The S-0026 session that authors this ADR installs the stack and verifies it runs.

- **Phase 4 is the first new code authored under all three layers.** The Phase 4 graph-construction work per [ADR 0016](0016-graph-construction-needs-live-validation.md) and [`../../build_plan/P_2_graph_validation.md`](../../build_plan/P_2_graph_validation.md) is the first session that exercises the contract on substantive new code (not retrofit). The graph-audit utility's `validate_graph()` arrives with a contract block, full pytest coverage, mypy strict cleanliness, and a cold-review pass at shutdown. Phase 3 (SQL Schema, [`../../build_plan/P_1_sql_schema.md`](../../build_plan/P_1_sql_schema.md)) authors mostly SQL migrations; the small Python footprint Phase 3 may carry runs under the contract from the first line.

- **Refinement under load is expected.** The first Phase 4 session running against this contract is the first real exercise. Friction surfaces, unclear contract-block conventions, gate-stack false positives — these are refinement signals, ENGINE_LOG-tracked, not contract failures. If the discipline produces friction disproportionate to its drift-prevention value, the contract gets revised under its own amendment discipline. The mechanism is calibrated to the compound-drift mode; if a different failure mode dominates, calibration changes.
