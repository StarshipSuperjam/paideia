### The work (S-0028)

The build-readiness report is the canonical contract. The summary below is for orientation; load the report for concrete column shapes, CHECK expressions, FK targets, and the rollback test sequence.

- **`users` mirror table** populated by trigger on `auth.users` insertion (T1-A); FK target for every learner-state cascade per amended [ADR 0031](../product/adr/0031-erasure-mechanism-and-individual-only-regime.md).
- **Tables to land** under `product/seed-graph/migrations/` per [`build_plan/P_1_sql_schema.md`](../build_plan/P_1_sql_schema.md):
  - `nodes` — id, label, domain[], summary, teaching_notes, aliases[], rigor_score_computed, rigor_score_adjustment, `confidence_level` (`CHECK ... IN ('EXTRACTED', 'INTERPRETED', 'SYNTHETIC')`), confidence, provenance, status (`CHECK ... IN ('active', 'deprecated', 'superseded')`), superseded_by UUID[], graph_version_added, timestamps.
  - `edges` — id UUID PRIMARY KEY DEFAULT gen_random_uuid(), source_id, target_id, type TEXT (unconstrained at schema; predicate validation at Phase 4), weight + confidence with `CHECK ... BETWEEN 0 AND 1`, provenance JSONB, evidence JSONB, graph_version_added, `UNIQUE (source_id, target_id, type)`.
  - `learner_events` — event-sourced log per [ADR 0015](../product/adr/0015-event-sourced-learner-model.md). Structured `context` columns (path_id, source_text_id, session_id) per [ADR 0026](../product/adr/0026-persistent-learner-storage-structural-not-substantive.md) (T1-E shape pinned in report). interaction_type CHECK + sub-signal range CHECK per T2-A, T2-B.
  - `mastery_snapshots` — cached mastery state per [ADR 0023](../product/adr/0023-engagement-depth-aggregation.md) / [ADR 0024](../product/adr/0024-engagement-depth-sub-signals-stored-raw.md) / [ADR 0025](../product/adr/0025-historical-maximum-tracking.md). PK `(user_id, concept_id)`; mastery_score, max_historical_score (DEFAULT 0), engagement_depth NULL, computed_at TIMESTAMPTZ; per-table indexes on user_id and concept_id (T2-C).
  - `tension_log` — per [`product/docs/self-correction.md`](../product/docs/self-correction.md) and [ADR 0026](../product/adr/0026-persistent-learner-storage-structural-not-substantive.md). `exchange_summary` JSONB with shape constraints enforcing the v1 vocabularies in [`TENSION_VOCABULARY.md`](../product/seed-graph/migrations/TENSION_VOCABULARY.md) (T1-C).
  - `settings` — singleton key/value table; graph_version initialized at 1 (T2-D); Phase 5 increment contract documented in ROUTING.md.
- **RLS on with v1 policies** (T1-B) — every learner-state table in `public.*` has `ENABLE ROW LEVEL SECURITY` plus `CREATE POLICY ... USING (user_id = auth.uid())` (or per-table equivalent). The pre-commit SQL-gates branch (Layer 2 of [ADR 0039](adr/0039-universal-expression-contract-across-ai-authoring-patterns.md)) hard-fails on any `CREATE TABLE public.*` without paired `ALTER TABLE ... ENABLE ROW LEVEL SECURITY`.
- **`ON DELETE CASCADE` discipline** per amended [ADR 0031](../product/adr/0031-erasure-mechanism-and-individual-only-regime.md): every FK to `users(id)` ends in `ON DELETE CASCADE`. The pre-commit SQL-gates branch hard-fails on missing CASCADE.
- **Deploy and verify** per the rollback test sequence in T2-H of the report.
