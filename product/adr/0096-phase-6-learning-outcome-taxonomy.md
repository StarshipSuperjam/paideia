# ADR 0096 — Phase 6 learning-outcome taxonomy: 3-level depth + Decoding-the-Disciplines bottleneck moves

- **Status:** Accepted
- **Date:** 2026-05-18
- **Deciders:** S-0203 (PDG papers extraction pre-phase Session δ₂; sibling ADR 0095 lands the tool-stack settlement in the same session)

## Context

[ADR 0094](0094-phase-6-scope.md) commits Phase 6 to expand to include Tier-A substrate redesign (Clusters 1-5). Cluster 3 — Goal-relative parameterization — introduces a new `learning_outcomes` table joined to `edges` via an `edge_outcome_necessity` junction table that carries `necessity_level`, `confidence_for_outcome`, and `rationale_for_outcome`. The Cluster 3 cluster-implementation ADR (Session δ₃+ or later) needs a settled outcome taxonomy to commit the `learning_outcomes` table's schema, the controlled vocabulary for `necessity_level`, and the structure of `rationale_for_outcome` entries.

[`engine/build_readiness/pdg_papers_extraction/synthesis.md`](../../engine/build_readiness/pdg_papers_extraction/synthesis.md) §Cluster 3 (L113-138) introduces the taxonomy at L123: "Outcome taxonomy at **minimum three levels** (introductory / analytical / comparative-research **as a starting taxonomy**)." [`Cluster 6`](../../engine/build_readiness/pdg_papers_extraction/synthesis.md) (Backward-design authoring, L208-229) reiterates: "Define target learning outcomes BEFORE drafting any nodes. Outcome taxonomy at minimum three levels (introductory / analytical / comparative-research)" (L218). The synthesis defaults to this 3-level depth taxonomy without citing a pedagogical framework — Bloom, SOLO, Decoding-the-Disciplines, Fink are not named.

This omission is load-bearing: a shallow outcome taxonomy produces weak goal-relative parameterization in Cluster 3, which in turn produces weak edge-outcome-necessity entries that bias downstream LLM-mediated teaching toward generic responses rather than discipline-specific operationalization. Per `adversarial_review.md` E.9.1 (lines 257-259) on goal-relativity: "outcome modifications trigger re-validation of `edge_outcome_necessity` entries" — meaning every taxonomy revision cascades into junction-table updates. The cost of choosing the wrong framework is non-trivial.

[`engine/build_readiness/pdg_papers_extraction/foundations.md`](../../engine/build_readiness/pdg_papers_extraction/foundations.md) F1 (Meyer & Land threshold concepts) establishes the threshold-concept framework already adopted by the project for `node_type: threshold_concept` (per Cluster 4). F3 (Middendorf & Pace bottleneck-vs-threshold distinction) establishes Decoding-the-Disciplines as the literature-grounded "where novices stuck" framework, with the bottleneck-as-omitted finding (the synthesis paper's enum lacks `bottleneck` as a node_type despite literature support). F3's subset-relationship finding: every threshold concept is a bottleneck (Middendorf & Pace 2004); but not every bottleneck is a threshold concept (the emotional/procedural bottleneck partition Pace & Middendorf 2011 names).

The session's choice is among (a) the synthesis default (3-level depth, no framework named); (b) Bloom or its Krathwohl 2002 revision; (c) SOLO; (d) Decoding-the-Disciplines bottleneck moves as the operational layer; (e) Fink's significant learning; (f) a hybrid that integrates the synthesis default with a literature-grounded framework. Per the quality-first deliberation posture from engine_memory pushback drawer [`1dc03200`](https://github.com/StarshipSuperjam/paideia), the hybrid that integrates the synthesis-paper-empirical default with the threshold-concept-and-bottleneck framework already implicit in the corpus is the choice that minimizes downstream re-decomposition while honoring the literature.

[ADR 0084](../../engine/adr/0084-pushback-rule-extraction-step-for-high-stakes-decisions.md) applies — this is a **contract-shape change**: the outcome taxonomy is the substrate Cluster 3's cluster-implementation ADR rests on, and the `rationale_for_outcome` column's vocabulary propagates as a foreclosing factor for the Cluster 3 ADR's schema choices. The Load-bearing premises subsection below dogfoods the extraction step. The premise count is lower than ADR 0095's (5 vs 8) because the choice is less foreclosing — a future revision of the taxonomy can amend this ADR without forcing migration re-decomposition, whereas ADR 0095's tool-stack decision is substrate-locking.

### Load-bearing premises

*(Per the [ADR 0084](../../engine/adr/0084-pushback-rule-extraction-step-for-high-stakes-decisions.md) extraction step.)*

1. **The 3-level depth granularity (introductory / analytical / comparative-research) is empirically adequate for parameterizing `edge_outcome_necessity` across Paideia's 9 philosophy subdomains.** *Falsifier:* Cluster 3 cluster-implementation ADR drafting discovers that 3 levels under-distinguish the edges actually needing differential parameterization — e.g., the gap between "analytical" and "comparative-research" turns out to span two discriminable necessity levels in practice (analytical-within-tradition vs analytical-across-traditions, both qualitatively below comparative-research). *Test status:* in-context partial test. The 380-node × 533-edge current graph has prerequisite chains averaging ~5-7 hops per subdomain (per S-0122 production-audit findings). 3 depth levels for a 5-7-hop chain gives ~2-hop granularity per level, which matches the empirical observation that Paideia's edges naturally cluster into discoverable / studyable / synthesizable bands. Synthesis paper case studies (per §Cluster 3 + §Cluster 6) use the 3-level scheme without escalation. **Premise verified within Phase-6-projected scale**; **named in Consequences as first-exercise readiness criterion T1-A** to close at the first Cluster 3 cluster-implementation ADR landing.
2. **Decoding-the-Disciplines bottleneck moves are a discipline-specific operationalization layer that integrates naturally with the threshold-concept framework already adopted via Cluster 4's `node_type: threshold_concept`.** *Falsifier:* Cluster 4's `node_type` enum (per Q1 settlement this session — multi-valued, concept-level) requires a bottleneck-moves operationalization that conflicts structurally with the 3-level depth scheme (e.g., the bottleneck-mental-action-modeling step belongs at the introductory level for some concepts and the analytical level for others, requiring per-edge bottleneck-tagging that the 3-level scheme does not encode). *Test status:* in-context verified via foundations.md F3 (Middendorf & Pace bottleneck-as-mental-action moves: bottleneck identification → mental-action modeling → modeling → practice/feedback → motivation → mastery assessment). Each bottleneck move maps onto one of the 3 depth levels: bottleneck identification + mental-action modeling at introductory; modeling + practice at analytical; motivation + mastery assessment at comparative-research. **Premise verified at authoring**; first-exercise readiness criterion not separately named — the integration is structural, not measured.
3. **Bloom's cognitive levels (and Krathwohl 2002 revision) are critiqued in the discipline-specific literature for being domain-agnostic in ways that produce shallow learning outcomes when imported as the canonical taxonomy.** *Falsifier:* the Krathwohl revision (remember / understand / apply / analyze / evaluate / create) is shown to be empirically equivalent to the 3-level depth + bottleneck-moves hybrid at Paideia's scale, AND the discipline-specific literature critiques are based on misapplications rather than structural domain-agnosticism. *Test status:* in-context verified via foundations.md F3 (Middendorf & Pace's Decoding-the-Disciplines work is specifically motivated by the inadequacy of Bloom for discipline-bottleneck identification) + the broader scholarly literature (Anderson & Krathwohl 2001's revision retains the cognitive-domain framing; later literature on disciplinary cognition consistently distinguishes from Bloom's hierarchy). The structural critique is that Bloom orders cognitive operations by complexity (remember → create) but does not distinguish how those operations differ across disciplines — applying-knowledge-in-physics is structurally different from applying-knowledge-in-history. **Premise verified.** The Decoding-the-Disciplines integration addresses the discipline-specificity gap that Bloom does not.
4. **SOLO's structural-complexity-of-single-learning-artifact framing (prestructural / unistructural / multistructural / relational / extended-abstract) does not naturally extend to the multi-concept goal-relative parameterization Cluster 3 needs.** *Falsifier:* SOLO's "relational" and "extended-abstract" levels can be re-interpreted as goal-relative parameterizations of multi-concept structures (e.g., "relational" = comparative-within-discipline; "extended-abstract" = comparative-across-disciplines) without loss of fidelity to Biggs & Collis's original construct. *Test status:* in-context partial test. SOLO is canonically about a single learning artifact's structural complexity (a student's response to one question, e.g.); applying it to edge-outcome-necessity requires re-framing the unit of analysis from artifact-response to edge-relationship. The re-framing is possible but loses the empirical anchoring SOLO has in the assessment-response literature. **Premise verified for the canonical SOLO interpretation;** the re-framed-SOLO option remains structurally viable but is rejected on grounds of integration cost rather than incompatibility (cf Alternatives Considered).
5. **The taxonomy is open-for-revision per the foundations.md §7.3 recommendation 2 pattern that ADR 0094 D8 will land separately;** future revisions amend this ADR via in-body subsection rather than supersede. *Falsifier:* a future revision proposes a structurally-different framework (e.g., switching from Decoding-the-Disciplines to a custom Paideia-specific bottleneck framework, or adding a fourth depth level that re-baselines the 3-level scheme) such that amendment is insufficient and full supersession is warranted. *Test status:* in-context verified at authoring. Adding new depth levels (e.g., "exploratory" below introductory; or "synthesizing" above comparative-research) is structurally additive and survives as amendment. Adding new bottleneck moves (e.g., "metacognitive-reflection" move) is structurally additive. Substituting a different bottleneck-resolution framework (e.g., problem-based learning's affordance structures) would be a structural change warranting supersession. **Premise verified for additive revisions;** supersession path preserved for structural framework changes.

## Decision

**Adopt synthesis C3's 3-level depth taxonomy (`introductory`, `analytical`, `comparative-research`) as the canonical Paideia learning-outcome depth taxonomy. Augment with Decoding-the-Disciplines bottleneck-resolution moves (per Middendorf & Pace 2004 and foundations.md F3) as the discipline-specific operationalization layer.**

The combined taxonomy carries two dimensions per outcome:

1. **Depth level** — one of `introductory` / `analytical` / `comparative-research`. Captures how far along a concept-mastery trajectory the outcome sits. Used as the `learning_outcomes.depth_level` column (likely a Postgres CHECK-constrained text column or enum); also used as a parameterization input to `edge_outcome_necessity.necessity_level` (for example, an edge required at the analytical depth may be `recommended` rather than `necessary` at the introductory depth).
2. **Bottleneck move** — one of `bottleneck-identification` / `mental-action-modeling` / `modeling` / `practice-feedback` / `motivation` / `mastery-assessment` (per Middendorf & Pace 2004's seven-step Decoding-the-Disciplines process, collapsed here to six discrete moves; the original seventh step "share resources" is operationally about teacher-collaboration rather than learner-mastery and falls outside Paideia's per-edge necessity model). Captures the discipline-specific operational shape of the outcome. Used as the `learning_outcomes.bottleneck_move` column (likely a Postgres CHECK-constrained text column).

`learning_outcomes` rows therefore carry: `(depth_level, bottleneck_move)` as a parameterizing tuple. The synthesis-paper's `rationale_for_outcome` column on the `edge_outcome_necessity` junction captures the per-edge narrative justification linking that edge to that outcome at that (depth_level, bottleneck_move) tuple.

**Threshold-concept integration is structural, not separate.** Cluster 4's `node_type` enum carries `threshold_concept` as one of its multi-valued enum members (per Q1 settlement this session). Threshold concepts are a special case of bottleneck per foundations.md F3 (every threshold concept is a bottleneck; not every bottleneck is a threshold concept). A learning outcome whose target node is a `threshold_concept` will typically carry the `mental-action-modeling` or `modeling` bottleneck move at the `analytical` or `comparative-research` depth level. The taxonomy does not enforce this structurally — it admits any (depth_level, bottleneck_move) for any target — but the discipline-authoring guidance (Cluster 6 — Backward-design authoring discipline, per Phase 7+ ops doc work) will recommend the threshold-concept → modeling-at-analytical-or-comparative-research pattern as the default.

**Open-for-revision per premise 5.** Additive revisions (new depth levels; new bottleneck moves; new bottleneck-domain combinations) amend this ADR via in-body subsection (mirrors ADR 0017 ← ADR 0095 in-body amendment pattern). Structural revisions (substituting a different bottleneck-resolution framework; restructuring the (depth_level, bottleneck_move) tuple into a different relationship) require full supersession.

**Per ADR 0094 dependency order C1 → C2 → C4 → C3 → C5, Cluster 3 lands fourth in the Tier-A sequence.** This ADR is the substrate for Cluster 3's cluster-implementation ADR; Cluster 3 cannot land until this ADR exists. The Cluster 3 cluster-implementation ADR will commit the `learning_outcomes` table schema using the (depth_level, bottleneck_move) tuple plus discipline-specific columns the cluster-implementation ADR will name (likely a `discipline` column joined to `nodes.disciplinary_domain` and an optional `rationale_template` JSONB column for structured rationale shapes).

## Alternatives Considered

### Synthesis 3-level depth + Decoding-the-Disciplines bottleneck moves (chosen)

- **What:** The hybrid above — 3 depth levels + 6 bottleneck moves as a parameterizing tuple on `learning_outcomes`.
- **Pros:** Honors the synthesis-paper's empirically-grounded 3-level depth scheme that case studies in §Cluster 3 + §Cluster 6 already use. Integrates with the threshold-concept framework already adopted via Cluster 4's `node_type` enum (foundations.md F1 / Meyer & Land 2003). Addresses the discipline-specificity gap that Bloom does not (premise 3). Structurally compatible with the multi-concept goal-relative parameterization Cluster 3 needs (premise 4 rejects re-framed-SOLO). Bounded vocabulary (3 × 6 = 18 cells; not all populated in practice) is operationally tractable. Open-for-revision (premise 5) preserves future flexibility.
- **Cons:** Two-dimensional vocabulary requires the Cluster 3 cluster-implementation ADR to commit a CHECK constraint structure for both columns + admit the cross-product. Decoding-the-Disciplines is a relatively narrow literature base (Middendorf & Pace's work + downstream practitioner publications); not as widely known as Bloom or SOLO, which means external contributors to OSS Paideia forks (per ADR 0093 commitment 2) may need pointer documentation to the source literature.
- **Rejected because:** not rejected — chosen.

### Synthesis 3-level depth taxonomy alone (no framework grounding)

- **What:** Adopt only the 3-level depth taxonomy without naming a discipline-specific operationalization layer.
- **Pros:** Simpler vocabulary (3 cells, not 18). No framework dependency.
- **Cons:** Leaves Cluster 3's `rationale_for_outcome` column under-grounded — the rationale entries have no controlled discipline-specific structure. Downstream LLM-mediated teaching responses risk being generic ("this edge supports analytical mastery") rather than operationally specific ("this edge supports the modeling bottleneck-move at analytical depth — students must internalize the formal-analyticity distinction before proceeding"). The synthesis paper's own omission of a framework (per Context above) is the gap; this option does not close it.
- **Rejected because:** under-grounds Cluster 3's `rationale_for_outcome` column; doesn't address the discipline-specificity gap.

### Bloom's taxonomy (Krathwohl 2002 revision: remember / understand / apply / analyze / evaluate / create)

- **What:** Adopt the Krathwohl revision as the canonical depth taxonomy. Replace the synthesis 3-level scheme.
- **Pros:** Widely known; high external recognition; substantial assessment-design literature; cognitive-domain hierarchy is intuitive for new contributors.
- **Cons:** Domain-agnostic in ways the discipline-specific literature critiques (premise 3). "Apply" in physics differs from "apply" in history in ways Bloom does not encode; this is the gap Decoding-the-Disciplines was developed to address. Six levels is finer-grained than Paideia's edge-outcome-necessity needs at projected scale (380 nodes; 533 edges) — finer granularity would force per-edge differentiation between e.g. "analyze-this-edge" vs "evaluate-this-edge" without empirical signal to support the distinction.
- **Rejected because:** discipline-agnostic; finer-than-needed granularity at Paideia's scale; the gap it leaves (discipline-specificity) is exactly the gap Decoding-the-Disciplines fills.

### SOLO (Biggs & Collis 1982: prestructural / unistructural / multistructural / relational / extended-abstract)

- **What:** Adopt SOLO as the canonical depth taxonomy. Replace the synthesis 3-level scheme.
- **Pros:** Structurally elegant; empirically anchored in assessment-response literature; the relational / extended-abstract levels capture the kind of cross-concept integration Paideia explicitly values (per ADR 0007 cross-domain porosity).
- **Cons:** Canonically about a single learning artifact's structural complexity, not about goal-relative parameterization of multi-concept-relationship edges (premise 4). Re-framing SOLO to fit Cluster 3's edge-outcome-necessity model is possible but loses empirical anchoring. Adds a fifth depth level (prestructural) that Paideia's edge-prerequisite model already encodes structurally (an edge exists or doesn't; "prestructural" with respect to the edge target is "edge not yet introduced") — redundant with the prerequisite-edge mechanism.
- **Rejected because:** unit-of-analysis mismatch (artifact-response vs edge-relationship); re-framing loses empirical anchoring; first depth level redundant with the prerequisite mechanism.

### Fink's significant learning (foundational / application / integration / human / caring / learning-how-to-learn)

- **What:** Adopt Fink's six dimensions as the canonical depth taxonomy. Replace the synthesis 3-level scheme.
- **Pros:** Honors non-cognitive dimensions of learning (human, caring) that Bloom and Decoding-the-Disciplines under-emphasize. Strong fit for whole-person learning frameworks.
- **Cons:** Broader-scope than C3's per-edge outcome necessity needs (premise 5 was about open-for-revision; this is about scope mismatch). The "human" and "caring" dimensions don't map naturally to per-edge parameterization — they are properties of the overall learning experience rather than of specific concept-to-concept dependencies. Fink himself describes the framework as a course-design tool, not an edge-relationship parameterization tool.
- **Rejected because:** scope mismatch (course-design tool vs per-edge parameterization); dimensions like "human" and "caring" don't map to per-edge necessity.

### Custom Paideia-specific outcome framework (build from scratch)

- **What:** Author a new outcome framework specific to Paideia's pedagogical-dependency-graph context. Don't import any existing literature framework.
- **Pros:** Maximum fit to Paideia's specific needs; no friction from literature-import.
- **Cons:** Violates the literature-grounding principle (per foundations.md §F2's "bridge_concept = coinage" finding — invented terms have lower discipline-rigor than literature-grounded ones). The synthesis paper's framework + Decoding-the-Disciplines integration already provides enough literature grounding for the hybrid to be defensible; inventing a new framework would require empirical validation that single-maintainer Paideia cannot supply at Phase 6+ scale.
- **Rejected because:** literature-grounding posture (cf foundations.md §F2); insufficient empirical-validation capacity at solo-maintainer scale.

## Consequences

The `necessity_level` enum on `edge_outcome_necessity` likely takes values like `required` / `recommended` / `optional` parameterized against the outcome's (depth_level, bottleneck_move); the Cluster 3 ADR will name the exact vocabulary.

- **No mechanism-first-exercise readiness note authored** per [ADR 0053](../../engine/adr/0053-mechanism-first-exercise-gate.md). The T1-A criterion above is a premise-disposition closure specific to this ADR's load-bearing premises.

- **The taxonomy is open-for-revision via in-body amendment** per premise 5.
