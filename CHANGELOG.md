# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) with project-specific calibration: changes are scoped to **state-of-record** (design docs, schemas, seed graph migrations, build_plan, ADRs, OPERATIONS-equivalent docs, top-level project files). Application code (React, API endpoints, UI) is **git-only** — its history lives in the commit log, not here.

This project does not yet follow [Semantic Versioning](https://semver.org/) — versions are milestones (`0.0.x` for foundation; `0.1.0` at foundation close; later versions track product readiness).

- [`adr/0030-confidence-level-evidentiary-mode-on-nodes.md`](adr/0030-confidence-level-evidentiary-mode-on-nodes.md) — `confidence_level` is the evidentiary-mode axis on nodes. Establishes the column as a third epistemic axis orthogonal to `provenance` (who authored, lifecycle stage) and to numeric `confidence` (how-sure-now, updates with evidence). The column records *what kind of evidence backed the node's existence at authoring time* and is fixed-at-write — supersession (per [ADR 0021](adr/0021-node-deprecation-status-superseded-by.md)) is the channel for changing it, not in-place update. Default `'INTERPRETED'` matches Phase 5.2 baseline; EXTRACTED would silently overclaim source backing on forgot-the-column writes; SYNTHETIC would over-flag the Phase 4 review queue. Two orthogonality test combinations recorded inline (`provenance='human'` + `confidence_level='SYNTHETIC'` for human-authored service nodes; `confidence=0.3` + `confidence_level='EXTRACTED'` for source-aligned nodes whose belief decayed under counter-evidence — the label does not update with that decay; the numeric belief does). Naming risk acknowledged: `confidence_level` reads as a finer version of `confidence` despite being a categorical evidentiary mode rather than a numeric belief — the name is settled across STATE.md / ROADMAP.md / ADR 0016 / Phase 5.2; renaming would invalidate downstream references; the ADR's Decision section is the durable mitigation. Storage representation: `TEXT` not Postgres ENUM, following `provenance` and `status` precedent — adding a new value is a superseding ADR + Phase 4 audit update, not a migration. Status: Accepted.
