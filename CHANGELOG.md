# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) with project-specific calibration: changes are scoped to **state-of-record** (design docs, schemas, seed graph migrations, build_plan, ADRs, OPERATIONS-equivalent docs, top-level project files). Application code (React, API endpoints, UI) is **git-only** — its history lives in the commit log, not here.

This project does not yet follow [Semantic Versioning](https://semver.org/) — versions are milestones (`0.0.x` for foundation; `0.1.0` at foundation close; later versions track product readiness).

### Added (S-0014 — Phase 1.5.2: Discovery / Planning / Engagement triad architecture)
- [`adr/0034-discovery-planning-engagement-triad.md`](adr/0034-discovery-planning-engagement-triad.md) — Discovery / Planning / Engagement triad as primary product structure. Replaces the globe-as-home-screen + concept-engagement-surface architecture retired by [ADR 0033](adr/0033-mission-realignment-structured-guidance-for-self-learners.md). Three primary surfaces, one each for the three structurally distinct problems the structured-guidance value claim answers: **Discovery** (figuring out what to learn at all — dual-path: AI conversational onboarding + browseable concept catalog; both committed first-class, neither a fallback for the other; both feed the same plan-generation pipeline downstream); **Planning** (knowing what to understand first — syllabus-as-plan, prerequisite gating, current/active syllabus view with the hard-cap-of-five preserved, quiet completion markers, mastered concepts visible in syllabus context, cross-syllabus convergence noting at the routing-after-concept-completion moment); **Engagement** (knowing whether you have actually learned the thing rather than become familiar with its facts — structurally inherited from [ADR 0028](adr/0028-input-side-scope-structural-not-prompt.md): three bounded contexts — concept engagement Mode 2/3, diagnostic conversation, BYOB close reading per [ADR 0011](adr/0011-no-hosted-copyrighted-material.md); preserved unchanged). Cross-domain edges remain first-class graph data per [ADR 0007](adr/0007-cross-domain-porosity.md); UI rendering convention is **bridge surfacing in context** during planning (cross-syllabus convergence notes; plan-generation overlap flagging) and during engagement (callback references that bridge between domains; in-engagement surfacing of cross-domain prerequisites that have just become reachable). Explicit foreclosures: no globe; no game-world rendering; no separate trophy surface; no library/bookshelf as a third primary surface (mastered concepts surface contextually within syllabi, not as a standalone trophy); no mastery-glow / tendril / colored-trail visualization; no general chat surface (per [ADR 0028](adr/0028-input-side-scope-structural-not-prompt.md)). Two open-for-amendment-without-supersession items flagged: library/bookshelf surface (if Phase 9 cold-test surfaces a concrete need for an aggregated mastered-concepts view consistent with the no-trophy-spectacle discipline) and discovery dual-path narrowing (if Phase 9 cold-test evidence makes the case for a single-path simplification). Discovery-side catalog organization may consume [ADR 0018](adr/0018-flat-domain-tags-community-detection.md)'s flat-domain-tags + community-detection algorithm as the concept-clustering primitive (the algorithm survives [ADR 0033](adr/0033-mission-realignment-structured-guidance-for-self-learners.md)'s globe drop because its consumer shifts; this ADR names the new consumer). Concurrent Syllabus Limit (hard cap of five) survives as a planning-surface constraint — the pedagogical rationale (insufficient momentum on more than five paths) survives; the visual-density-of-the-globe-itself signaling rationale dropped with the globe; the planning surface signals the cap directly when the user attempts to exceed it. Bidirectional with [ADR 0027](adr/0027-rendering-policy-prompt-layer-contract.md)'s pending S-0016 forbidden-token revision: the surface around the agent does not invite game-master framing (no game-shaped surfaces in the triad), and the agent's voice does not reach for that framing (the rendering policy will rule it out). Does not rise to MISSION.md's strong-working-commitments list (product-structural, not pedagogical commitment — same precedent as ADRs 0026/0029/0031/0032/0033). Future feature design proposing a primary surface outside the triad must supersede this ADR; secondary surfaces (settings, account management, billing, delete-account, data-export, parked-syllabus list, user library of uploaded texts) do not require supersession. Status: Accepted.

### Added (S-0010 — Phase 1.3: `confidence_level` column on Node Schema; closes Phase 1 modulo OQ-PRIVACY-A/B)
- [`adr/0030-confidence-level-evidentiary-mode-on-nodes.md`](adr/0030-confidence-level-evidentiary-mode-on-nodes.md) — `confidence_level` is the evidentiary-mode axis on nodes. Establishes the column as a third epistemic axis orthogonal to `provenance` (who authored, lifecycle stage) and to numeric `confidence` (how-sure-now, updates with evidence). The column records *what kind of evidence backed the node's existence at authoring time* and is fixed-at-write — supersession (per [ADR 0021](adr/0021-node-deprecation-status-superseded-by.md)) is the channel for changing it, not in-place update. Enum semantics: **EXTRACTED** (lifted from a source's own structuring — SEP TOC, curriculum prerequisite list, graph dataset; vocabulary alignment not concept invention), **INTERPRETED** (pedagogical inference from source material that does not itself structure around the concept; Phase 5.2 baseline per "mark INTERPRETED until validated"), **SYNTHETIC** (generated wholesale; service nodes terminating cross-domain prerequisite chains, Opus-batch-proposed splits without source basis; first candidates for self-correction review). Default `'INTERPRETED'` matches Phase 5.2 baseline; EXTRACTED would silently overclaim source backing on forgot-the-column writes; SYNTHETIC would over-flag the Phase 4 review queue. Two orthogonality test combinations recorded inline (`provenance='human'` + `confidence_level='SYNTHETIC'` for human-authored service nodes; `confidence=0.3` + `confidence_level='EXTRACTED'` for source-aligned nodes whose belief decayed under counter-evidence — the label does not update with that decay; the numeric belief does). Naming risk acknowledged: `confidence_level` reads as a finer version of `confidence` despite being a categorical evidentiary mode rather than a numeric belief — the name is settled across STATE.md / ROADMAP.md / ADR 0016 / Phase 5.2; renaming would invalidate downstream references; the ADR's Decision section is the durable mitigation. Storage representation: `TEXT` not Postgres ENUM, following `provenance` and `status` precedent — adding a new value is a superseding ADR + Phase 4 audit update, not a migration. Status: Accepted.

### Added (S-0006 — Phase 1.1: prompt-pack Session 11 — Historical Maximum Tracking)
