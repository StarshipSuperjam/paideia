### A2 — MemPalace capture wrapper

[`.claude/settings.json`](.claude/settings.json) configures `Stop` and `PreCompact` hooks as fire-and-forget `mempalace hook run` commands. If the binary is missing or capture errors, the hook fails silently — exploration sessions can be lost from MemPalace without any signal. The user-approved fix (option a per the plan) is a wrapper script that surfaces failures to a session-visible log file the AI can spot at next boot.

Implementation:
- New file `tools/hooks/mempalace-hook-wrapper.sh` (tracked, executable). The wrapper accepts the same `--hook` and `--harness` arguments, invokes `mempalace hook run` with them, captures exit code + stderr, and appends a timestamped success/failure line to `.claude/logs/mempalace-hook.log` (gitignored, per-environment).
- Update [`.claude/settings.json`](.claude/settings.json) `Stop` and `PreCompact` hooks to invoke the wrapper instead of `mempalace hook run` directly.
- The wrapper never blocks the harness — it always exits 0 to the harness regardless of `mempalace`'s actual exit code; the log carries the truth.
- Session-boot procedure (in CLAUDE.md or the build-lifecycle Recovery section, depending on where it fits cleanest): if `.claude/logs/mempalace-hook.log` shows failure entries since last boot, surface them to the user.

### B4 — ADR index/file consistency validator extension

[`tools/validate.py`](tools/validate.py) currently does not check whether every file in `adr/` is listed in [`adr/README.md`](adr/README.md)'s per-phase tables, or whether the Status field in each ADR file matches the index entry. Discipline so far has held (numbering 0001-0036 clean, statuses consistent), so this is a defensive check — soft-warn rather than hard-fail because the index is human-curated and edge cases (Superseded entries with multi-line status notes, tables with formatting variation) need judgment.
